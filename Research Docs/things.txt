"The dashboard originally made N+1 database queries — one per agent for tasks and incidents. With 50 agents that's 101 queries on every page refresh. I fixed it by fetching all tasks and incidents upfront in two batch queries, then grouping them by agent_id in memory using a dictionary lookup. Now it's always 3 queries regardless of fleet size. The tradeoff is slightly higher memory usage but for a C2 dashboard that's completely acceptable."
